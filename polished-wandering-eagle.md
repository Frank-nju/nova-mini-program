# 《追光健雄｜云端数字展馆》项目计划

## 一、项目定位与核心理念

### 产品名称
**《追光健雄｜云端数字展馆》**

### 产品愿景
打造一款**兼具强视觉冲击力、高频互动性与深厚文化底蕴的数字展览馆级小程序**，展示同学们共读《吴健雄传》的成果作品，弘扬科学家精神。

### 四大核心创新模块

| 模块 | 创新点 | 落地难度 | 替代方案 |
|------|--------|---------|---------|
| **沉浸式展厅** | 3D虚拟展馆漫游 | 高 | 伪3D视差滚动 / 全景滑动长页 |
| **跨时空对话** | AI吴健雄专属对话 | 中 | RAG知识库 + 大模型API ✅ |
| **数字文创集章** | 精神印记徽章系统 | 低 | Canvas绘制海报 ✅ |
| **多媒体成果矩阵** | 高质量作品展陈 | 低 | 瀑布流 + 视频优化 ✅ |

## 二、核心功能详细设计

### 1. 沉浸式展厅（创新视觉方案）

#### 方案A：伪3D视差滚动展厅（推荐，可落地）
**技术原理**：多层背景以不同速度滚动，产生深度视差效果

```
视觉层级设计：
┌─────────────────────────────────┐
│  Layer 1: 星空/粒子背景（最慢）    │  ← Canvas粒子动画
├─────────────────────────────────┤
│  Layer 2: 吴健雄剪影轮廓（中速）   │  ← SVG动态轮廓
├─────────────────────────────────┤
│  Layer 3: 作品卡片（正常速度）     │  ← 瀑布流布局
├─────────────────────────────────┤
│  Layer 4: 前景装饰粒子（最快）     │  ← CSS动画
└─────────────────────────────────┘
```

**实现要点**：
- CSS `transform: translateZ()` + `perspective` 创建视差
- Canvas绘制动态星空/物理粒子背景
- 滚动触发 Intersection Observer 动画
- 作品卡片进场动画（淡入+缩放+位移）

#### 方案B：全景滑动长页（更易实现）
**设计理念**：博物馆长卷式浏览体验

```
页面结构：
┌──────────────────────┐
│  封面动画（粒子聚合成肖像）  │
├──────────────────────┤
│  第一展区：生平履迹         │  ← 时间轴+里程碑卡片
│  ├─ 1912-1936 求学时期     │
│  ├─ 1936-1956 赴美深造     │
│  └─ 1956-1997 归国交流     │
├──────────────────────┤
│  第二展区：治学风骨         │  ← 名言卡片+故事插图
├──────────────────────┤
│  第三展区：科研丰碑         │  ← 实验动画+成果图解
├──────────────────────┤
│  成果展示墙（同学作品瀑布流）  │
├──────────────────────┤
│  AI对话入口              │
├──────────────────────┤
│  集章成就页              │
└──────────────────────┘
```

#### 方案C：交互式时间轴展厅
**设计理念**：横向滑动时间轴 + 点击展开详情

```
交互方式：
←─────────────────────────────────────→
   1912         1936         1956
    ○────────────○────────────○
    │            │            │
  [出生]       [赴美]      [宇称实验]
    ↓            ↓            ↓
  展开详情     展开详情      展开详情
```

### 2. 跨时空对话（AI赋能）

#### 技术架构

```
用户提问 → 前端 → FastAPI后端 → RAG检索《吴健雄传》知识库
                                    ↓
                              Claude/Kimi API
                                    ↓
                              流式回答 → 前端渲染
```

#### RAG知识库构建
1. **语料清洗**：提取《吴健雄传》全文，按章节/主题分段
2. **向量化存储**：使用文本嵌入模型（如 text-embedding-3-small）
3. **检索增强**：用户提问时，先检索相关段落，再作为上下文给大模型

#### Prompt工程设计
```
System Prompt:
你是吴健雄先生，一位杰出的核物理学家。请基于以下知识回答用户问题：

[检索到的相关段落]

回答要求：
1. 以吴健雄先生的第一人称口吻回答
2. 温和亲切，善用比喻解释科学概念
3. 体现严谨求实的治学态度
4. 适当引用自己的人生经历
5. 鼓励年轻人追求科学真理
```

#### 预设问题快捷入口
- "吴先生，您是如何发现宇称不守恒的？"
- "做科研最重要的是什么？"
- "您对中国科学发展有什么期望？"
- "您对年轻人有什么寄语？"

### 3. 数字文创集章系统

#### 徽章设计（精神印记）

| 徽章名称 | 解锁条件 | 设计元素 |
|----------|----------|----------|
| **追光者** | 完成首页浏览 | 光芒/追寻符号 |
| **求知者** | 阅读任意3篇心得 | 书卷/知识符号 |
| **探索者** | 浏览全部三大展区 | 指南针/探索符号 |
| **对话者** | 完成首次AI对话 | 对话气泡符号 |
| **传承者** | 分享作品到朋友圈 | 传递/火炬符号 |
| **健雄精神** | 集齐以上5枚徽章 | 吴健雄剪影+光芒 |

#### 海报生成技术
- Canvas绘制个性化分享海报
- 包含用户头像、已解锁徽章、专属寄语
- 支持保存到相册、分享朋友圈

### 4. 多媒体成果矩阵

#### 作品类型展示

| 作品类型 | 展示方式 | 交互特性 |
|----------|----------|----------|
| **读书心得** | 富文本阅读页 | 点赞/评论 |

- 图片WebP格式 + 懒加载

- 长列表虚拟滚动

## 五、技术选型

### 前端技术栈

| 技术 | 选型 | 理由 |
|------|------|------|
| 框架 | **Uni-app (Vue3)** | 跨端、生态成熟、性能好 |
| UI库 | **uView UI 3.0** | 组件丰富、文档完善 |
| 状态管理 | **Pinia** | Vue3官方推荐 |
| 动画 | **GSAP + CSS3 + Canvas** | 高性能动画方案 |
| 瀑布流 | 自定义组件 | 更灵活控制 |

### 后端技术栈

| 技术 | 选型 | 理由 |
|------|------|------|
| 服务 | **微信云开发** | 免运维、天然集成微信生态 |
| 数据库 | **云数据库** | NoSQL、灵活扩展 |
| 存储 | **云存储 + CDN** | 高效分发多媒体 |
| AI | **Claude API** | 对话质量最佳 |

### AI技术方案

| 组件 | 选型 |
|------|------|
| 大模型 | Claude 3.5 Sonnet / Kimi |
| Embedding | text-embedding-3-small |
| 向量库 | 云端存储 / FAISS |
| 流式输出 | Server-Sent Events |

## 七、致胜关键Tips

1. **沉浸式视差展厅**：伪3D视差滚动 + Canvas粒子动画，打造博物馆级浏览体验

3. **思政教育数字化**：让思政内容以年轻人喜闻乐见的方式呈现

### 答辩话术要点

1. **创新性**："我们不是简单的内容展示，而是打造了一个'云端数字展馆'"
2. **技术深度**："RAG知识库、视差动画、Canvas粒子系统..."
3. **用户体验**："沉浸式浏览、AI互动、游戏化集章..."
4. **社会价值**："弘扬科学家精神、思政教育数字化创新"

## 八、风险与应对

| 风险 | 应对措施 |
|------|----------|
| 视差动画性能问题 | 降级方案：关闭动画、简化效果 |
| AI回答幻觉 | 敏感词过滤 + 人工审核预设回答 |
| 素材质量参差不齐 | B同学统一精修排版 |
| 开发进度延期 | 核心功能优先，创新功能可裁剪 |
| 小程序审核不通过 | 提前了解审核规范，预留修改时间 |
