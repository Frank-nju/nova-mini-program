# 《追光健雄｜云端数字展馆》项目计划（深化版）

## 一、项目定位与核心理念

### 产品名称
**《追光健雄｜云端数字展馆》**

### 产品愿景
打造一款**兼具强视觉冲击力、高频互动性与深厚文化底蕴的数字展览馆级小程序**，展示同学们共读《吴健雄传》的成果作品，弘扬科学家精神。
最终落地形态采用"**滚动卷轴式场馆长页 + 数字人导览 + 事件云图回看 + 分阶段勋章反馈**"的统一叙事路径，让用户以参观展馆的节奏逐步理解吴健雄先生的人生事迹、科研贡献与精神传承。

### 四大核心创新模块

| 模块 | 创新点 | 落地难度 | 替代方案 |
|------|--------|---------|---------|
| **沉浸式展厅** | 滚动卷轴式场馆长页定稿 | 中高 | 伪3D视差分层 + 事件云图导航 ✅ |
| **跨时空对话** | 数字人导览 + AI吴健雄专属对话 | 中 | RAG知识库 + 大模型API ✅ |
| **数字文创集章** | 分阶段精神印记勋章系统 | 低 | Canvas绘制海报 ✅ |
| **多媒体成果矩阵** | 事迹穿插式高质量作品展陈 | 低 | 多类型适配容器 + 视频优化 ✅ |

## 二、核心功能详细设计

### 1. 沉浸式展厅（创新视觉方案）

**方案定稿**：最终采用"**滚动卷轴式场景化长页**"作为唯一主展示形态。方案A负责视觉表现，方案B提供长卷叙事骨架，方案C下沉为卷轴中的节点交互能力，三者合并为一条连续体验链路，不再作为并列候选方案。

#### 1.1 方案A：伪3D视差滚动展厅（推荐，可落地）
**技术原理**：多层背景以不同速度滚动，产生深度视差效果，并通过长卷式分区推进展厅叙事

```
视觉层级设计：
┌──────────────────────────────────────┐
│  Layer 1: 星空/粒子背景（最慢）    │  ← Canvas粒子动画
├──────────────────────────────────────┤
│  Layer 2: 吴健雄剪影轮廓（中速）   │  ← SVG动态轮廓
├──────────────────────────────────────┤
│  Layer 3: 事迹节点+成果卡片（正常速度）│  ← 卷轴分镜布局
├──────────────────────────────────────┤
│  Layer 4: 前景装饰粒子（最快）     │  ← CSS动画
└──────────────────────────────────────┘
```

**实现要点**：
- CSS `transform: translateZ()` + `perspective` 创建视差
- Canvas绘制动态星空/物理粒子背景
- 滚动触发 Intersection Observer 动画
- 作品卡片进场动画（淡入+缩放+位移）
- 每个卷轴段落固定"事迹摘要 + 代表引语 + 对应成果入口 + 数字人讲解提示"
- 卷轴尾段接入事件云图总览、AI问答入口与勋章结算反馈，形成完整收束
- 采用"强叙事段落 + 轻交互停顿 + 成果穿插展陈"的节奏，避免长页浏览疲劳

#### 1.2 方案B：全景滑动长页（更易实现）
**设计理念**：作为最终方案的叙事骨架，提供博物馆长卷式浏览体验

```
页面结构：
┌──────────────────────────────────┐
│  封面动画（粒子聚合成肖像）  │
├──────────────────────────────────┤
│  第一展区：生平履历         │  ← 时间轴+里程碑卡片+心得摘录
│  ├─ 1912-1936 求学时期     │
│  ├─ 1936-1956 赴美深造     │
│  └─ 1956-1997 归国交流     │
├──────────────────────────────────┤
│  第二展区：治学风骨         │  ← 名言卡片+故事插绘+海报作品
├──────────────────────────────────┤
│  第三展区：科研丰碑         │  ← 实验动画+成果图解+视频/PPT
├──────────────────────────────────┤
│  事件云图总览              │
├──────────────────────────────────┤
│  AI对话入口              │
├──────────────────────────────────┤
│  集章成就页              │
└──────────────────────────────────┘
```

**卷轴叙事推进方式**：
- **序章开场**：以粒子聚合肖像、题字与数字人欢迎词建立"进入展馆"的第一感受
- **生平履历段**：按时间推进关键人生节点，同步插入求学笔记、人物卡与读书心得短摘
- **治学风骨段**：用名言卡、手稿局部、同学海报与漫画穿插展示其人格力量
- **科研丰碑段**：以实验原理图、视频讲述、PPT拆页和成果图解解释科学贡献
- **尾厅收束段**：汇总事件云图、引导AI问答、展示勋章进度与个人学习成果

#### 1.3 方案C：交互式时间轴展厅
**设计理念**：作为卷轴中的嵌入式节点控件，提供横向滑动时间轴 + 点击展开详情的补充交互

```
交互方式：
←──────────────────────────────────────→
   1912         1936         1956
    ○────────────○────────────○
    │            │            │
  [出生]       [赴美]      [宇称实验]
    ↓            ↓            ↓
  展开详情     展开详情      展开详情
```

**交互补充说明**：
- 时间轴不再独立成馆，而是嵌入"生平履历"与"科研丰碑"两大展区中，作为关键节点放大器
- 点击节点后优先弹出"事件摘要 + 代表成果 + 数字人语音讲解入口"，保证叙事不断裂
- 卷轴任意关键节点均可跳转到尾厅事件云图，完成"线性浏览"与"非线性总览"的双向呼应

### 1.4 【深化】素材分类整理流程

#### 素材分类台账设计（四维分类系统）

| 维度 | 字段名 | 可选值 | 示例 |
|-----|-------|-------|------|
| 内容主题 | `theme` | 生平履历/治学风骨/科研丰碑 | `theme: "生平履历"` |
| 作品类型 | `contentType` | article/ppt/poster/video/comic/h5 | `contentType: "video"` |
| 素材状态 | `status` | draft/review/online/revision/archive | `status: "review"` |
| 适配位置 | `slot` | scroll_embed/tail_gallery/cloud_jump/dh_recommend | `slot: "scroll_embed"` |
| 节点绑定 | `nodeBinding` | 具体事迹节点ID | `nodeBinding: "node_1936_usa"` |

**命名规范**：
```
格式: {展区}_{节点}_{类型}_{作者}_{版本}_{日期}
示例: 生平履历_赴美深造_video_张三_v1_20250401
```

#### 素材入库流程（6步）

```
Step 1: 素材收集
  ├─ 来源：书院共读活动收集、同学提交、导师整理
  └─ 输出：原始素材文件夹

Step 2: 初筛归类
  ├─ 执行者：成员B（视觉创意与内容排版师）
  ├─ 动作：填写分类台账，确定theme/contentType/nodeBinding
  └─ 输出：素材台账CSV/Excel

Step 3: 精修排版
  ├─ 执行者：成员B
  ├─ 动作：图片精修、视频剪辑、排版统一化
  └─ 输出：标准化素材文件

Step 4: 内容审核
  ├─ 执行者：导师/组长
  ├─ 动作：检查内容准确性、精神内核匹配度
  └─ 输出：审核通过标记

Step 5: 适配容器
  ├─ 执行者：成员D（前端业务开发）+ 成员B
  ├─ 动作：确定展示方式（scroll_embed/tail_gallery等），前端适配
  └─ 输出：适配代码+测试链接

Step 6: 上线入库+数字人关联
  ├─ 执行者：成员E（后端）+ 成员F（AI开发）
  ├─ 动作：数据库写入+数字人推荐语绑定
  └─ 输出：正式上线状态
```

#### 状态流转说明

```
draft（待整理）
  ↓ 初筛归类完成
review（待审核）
  ↓ 导师/组长审核通过
online（可上线）
  ↓ 发现问题需修改
revision（需返修）
  ↓ 修改完成重新审核
online（可上线）
  ↓ 不再使用
archive（已归档）
```

### 1.5 【深化】多类型文件兼容容器

| 类型 | 容器组件 | 展示方式 | 返回逻辑 | 数字人联动 |
|-----|---------|---------|---------|-----------|
| 文章(心得) | `<RichTextReader />` | 展开卡片→富文本页 | 返回卷轴锚点 | 悬浮推荐语气泡 |
| PPT | `<SlideViewer />` | 幻灯片预览窗 | 关闭返回 | 解说音频入口 |
| 海报 | `<GallerySlider />` | 横向画廊 | 缩小返回 | 作者介绍浮层 |
| 视频 | `<VideoPlayer />` | 封面+播放器 | 退出返回 | 观看计数+推荐 |
| 漫画/长图 | `<LongImageViewer />` | 缩略图→全屏滑动 | 返回原位 | 章节导航提示 |
| H5 | `<WebViewEmbed />` | 入口按钮→全屏WebView | 返回锚点 | 数字人引导语 |

**前端实现要点**：
- 每个容器组件必须接收 `anchorId` 参数，用于返回定位
- 容器组件统一暴露 `onClose` 事件，触发卷轴滚动回原位
- 使用 `IntersectionObserver` 懒加载容器内容
- 卷轴内优先加载首屏与当前节点素材，云图、H5与高清视频按需二次拉取

### 1.6 【深化】事件云图总览模块

#### 云图节点类型与视觉编码

#### 云图层级结构

```
中心: 吴健雄肖像（Level 0）

第一环(Level 1): 人生阶段节点
  ├─ 1912-1936 求学时期
  ├─ 1936-1956 赴美深造
  └─ 1956-1997 归国交流

第二环(Level 2): 科研事件节点
  ├─ 宇称不守恒实验(1956)
  ├─ β衰变研究

  ├─ 杨振宁、李政道（合作）
  ├─ "严谨求实"、"独立思考"（关键词）
  └─ 同学作品节点（成果）
```

#### 云图交互模式

| 模式 | 解锁条件 | 交互方式 |
|-----|---------|---------|
| 线性浏览 | 默认开启 | 随卷轴进度逐步点亮节点 |
| 自由探索 | 获得"健雄精神"勋章 | 点击任意节点展开详情 |
| 答辩演示 | 解锁后+演示模式 | 作为全局导航页 |

**点击节点行为**：
```
节点点击 → 弹出详情卡 → 数字人讲解音频 → 跳转按钮（到卷轴/到成果）
    ↓          ↓              ↓                  ↓
```

### 2. 跨时空对话（AI赋能）

#### 技术架构

```
用户滚动/点击 → 数字人导览层 → 前端 → FastAPI后端 → RAG检索《吴健雄传》知识库
                                                  ↓
                                            Claude/Kimi API
                                                  ↓
                             流式回答/讲解气泡/勋章反馈/成果推荐 → 前端渲染
```

#### RAG知识库构建
1. **语料清洗**：提取《吴健雄传》全文，按章节/主题分段
2. **向量化存储**：使用文本嵌入模型（如 text-embedding-3-small）
3. **检索增强**：用户提问时，先检索相关段落，再作为上下文给大模型
4. **节点标注**：为每段语料增加"展区位置、事件节点、精神关键词、关联成果类型"标签，便于数字人在不同场景切换讲解角色

#### Prompt工程设计
```
System Prompt:
你是吴健雄先生，一位杰出的核物理学家。请基于以下知识回答用户问题：

[检索到的相关段落]

回答要求：
1. 以吴健雄先生的第一人称口吻回答
2. 温和亲切，善用比喻解释科学概念
3. 体现严谨求实的治学态度
4. 适当引用自己的人生经历
5. 鼓励年轻人追求科学真理
6. 若用户正处于某个展区，则优先结合当前节点讲解并推荐对应成果内容
```

#### 预设问题快捷入口
- "吴先生，您是如何发现宇称不守恒的？"
- "做科研最重要的是什么？"
- "您对中国科学发展有什么期望？"
- "您对年轻人有什么寄语？"

### 2.1 【深化】数字人深度融合设计

#### 数字人六种角色状态定义

| 状态 | 触发场景 | 视觉表现 | 语音时长 | 示例内容 |
|-----|---------|---------|---------|---------|
| **欢迎态(welcome)** | 首次进入首页 | 站立+手势欢迎 | 20s | "欢迎来到云端数字展馆..." |
| **讲解态(explain)** | 点击展区讲解按钮 | 半身+手势指向 | 30-60s | "这一段讲的是我赴美深造..." |
| **推荐态(recommend)** | 浏览作品卡片时 | 头像浮现+提示气泡 | 5-10s | "这件海报与我的求学经历呼应..." |
| **反馈态(feedback)** | 解锁勋章时 | 微笑+祝贺手势 | 15s | "恭喜你获得风骨共鸣勋章..." |
| **问答态(qa)** | 进入AI对话入口 | 坐姿+倾听姿态 | 动态 | RAG检索生成 |
| **总结态(summary)** | 完成全部成就后 | 庄重站立 | 30s | "你已完整理解了健雄精神..." |

| 模块 | 联动点 | 数字人动作 | 技术实现 |

| AI问答 | 任意页面底部入口 | 切换qa态 | `<router-link to="/qa"><DHButton /></router-link>` |

### 3. 【深化】数字文创集章系统

#### 徽章设计（精神印记）

成就触发拆分为**事迹浏览、成果观看、互动参与**三条链路，勋章既记录用户看到了哪里，也反馈其理解到了什么。

| 徽章名称 | 触发类型 | 解锁条件 | 设计元素 | 精神映射 |
|----------|----------|----------|----------|----------|
| **启程之光** | 事迹浏览 | 完成首页导览与序章浏览 | 光芒/门扉 | 对科学道路的主动靠近 |

#### 作品类型展示

| 作品类型 | 展示方式 | 交互特性 | 适配位置 |
|----------|----------|----------|----------|
| **读书心得** | 富文本阅读页 | 点赞/评论/摘录收藏 | 生平履历节点后的延伸阅读卡 |

| **漫画/长图** | 长图查看 | 缩放/滑动/章节定位 | 生平履历与治学风骨之间的转场段 |

#### 多类型适配规则

- 文章类适合承接节点后的延伸阅读，默认收纳为展开卡片，避免破坏卷轴节奏
- 图片与海报类适合做视觉停顿点，采用横向滑看或画廊放大，强化"展陈感"
- 视频与PPT类优先用于解释复杂科研内容，入口放在"科研丰碑"节点附近，并提供继续阅读按钮
- 长图、漫画和H5类适合作为转场或专题展开内容，需要保留返回卷轴原位置的锚点，保证体验连续

#### 性能优化
- 图片WebP格式 + 懒加载
- 视频预览图 + 按需加载
- 分包加载（主包<2MB）
- 长列表虚拟滚动
